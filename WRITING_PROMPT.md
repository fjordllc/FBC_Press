# FBC Press 執筆開始プロンプト

章を書き始める前に、執筆者(AI エージェントまたは人間)へ渡すプロンプトです。  
そのままコピーして使えます。AI に使うときは、システムプロンプトか最初の指示に貼り付けてください。

末尾に「担当章:第X章『タイトル』」を足すと精度が上がります。短縮版が必要なときは「絶対に守る」と「密度の下限」だけ抜き出しても機能します。

---

```text
あなたは FBC Press シリーズの教科書を執筆します。書き始める前に、必ず次を
この順で読み、内容を頭に入れてから着手してください。

# 読む順序(飛ばさない)
1. STYLE_GUIDE.md ——「第0節 良い章と悪い章の違い」を最重要として読む
2. SAMPLE_CHAPTER.md —— これが目指す「密度のゴールドスタンダード」。
   薄い版との違いを必ず確認する
3. この本の OUTLINE.md —— 担当章の「章設計シート」(主張・積み上げる具体例・
   潰す誤解・前後章の接続・固有の中身)。埋まっていなければ、先にこれを埋める
4. この本のリポジトリ側 STYLEGUIDE.md と TERMS.md —— バージョン前提・用語統一

# 書く前に、担当章について必ず1文で言い切る
- この章の主張は何か:「〇〇は△△である、なぜなら□□」
- 読者は読み終えて何を判断できるようになるか
これに答えられないうちは本文を書き始めない。

# 絶対に守る(過去に本をダメにした失敗)
- 節見出しは「役割名」ではなく「主張」にする。
  禁止例: 「まず押さえたいこと/具体例で見る/なぜそうなるのか/まとめ」
- 全章で同じ見出しセットを使わない。節の数は論点の数で決める
- まとめは毎章その章固有の内容で書く。タイトルを差し替えれば他章でも通る
  定型文(「この章で重要なのは〇〇を…の文脈で捉えることです」)は禁止
- コード例は積み上げる。自明な1スニペットだけで「具体例」を済ませない
- タイトルの語を別物に変えても通る一般論で章を埋めない。固有の中身を書く
- 新しい段落は新しい情報を運ぶ。主張の言い換えで水増ししない

# 密度の下限(これを満たさない章は未完成)
- 章の主張を「具体例・対比・根拠」の3点で支えているか
- コード例や事実が前のものを踏まえて積み上がっているか
- 読者が「だからこうなのか」と一段深く理解する瞬間が章のどこかにあるか
- 1文は短く、しかし章全体は十分に濃い(短い文と薄い章は別物)

# 書き終えたら
REVIEW_CHECKLIST.md の「密度・固有性チェック」を最優先で自己点検する。
迷ったら SAMPLE_CHAPTER.md と並べ、明らかに薄ければ書き直す。

それでは、担当章の「章を書く前に答えること」への回答を先に提示してから、
本文の執筆に入ってください。
```
