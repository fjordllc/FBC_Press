# FBC Press Asset Rules

この文書は、図版、画像、補助アセットの管理ルールを定義します。

## 基本方針

- アセットは原稿の一部として扱う
- 出典、生成元、用途が分からないファイルを置かない
- 同じ図の派生物を無秩序に増やさない
- 本文との対応関係がファイル名で分かるようにする

## 図版ファイル

- 可能なら SVG を優先する
- ファイル名は `fig-章-図番号.svg` に統一する
- 本文の図番号と一致させる
- 図中の文言は本文の用語に合わせる
- 出力先は `manuscript/assets/` に統一する（新規・改修時）。旧来 `manuscript/figures/` に置いている本は、集約のタイミングで `manuscript/assets/` へ寄せ、本文の参照パスも合わせて直す。
- 図番号やキャプション文を SVG 内に焼き込まない。番号は本文とファイル名で対応させる（`FIGURE_GUIDE.md`「図番号とキャプションの置き場所」）。

## 画像ファイル

- 写真やビットマップが必要なときだけ PNG/JPEG を使う
- スクリーンショットは用途を明確にする
- 解像度不足の画像を置かない

## 第三者の資料画像とライセンス

- 写真・スクリーンショットなど第三者が作った画像を使うときは、**ライセンスを必ず確認する**。
- パブリックドメインか、再利用が許された CC などのライセンスのものだけを使う。**CC BY / CC BY-SA は表示義務**(BY-SA はさらに継承義務)があるので、作者名と出典を明記する。
- ライセンスが確認できない画像は使わない。「たぶん大丈夫」で載せない。
- 図解は、権利が確実な**自作の SVG を優先**する。第三者画像のライセンス管理そのものを避けられる。
- CC BY-SA のような継承義務つきの画像は、本のライセンス(例: MIT)と衝突しうる。採用前に本のライセンスとの整合を確認する。
- 使った画像は、出典・作者・ライセンス・必要な帰属表示を、クレジット欄(奥付や付録)に残す。

## 生成元の管理

- AI 生成、デザインツール、手描き、スクリーンショットなど、作成手段を把握する
- 再生成が必要な図は、元データまたは再現手順を残す
- 後から差し替えできない単発ファイルを増やしすぎない
- **自作 SVG を多数作る本では、生成スクリプトを 1 本に集約する**(例: `scripts/figures/build_all.py`)。図ごとにばらばらの手順を残さず、1 スクリプトで全図を再生成できる状態にする。配色・字形・余白などの共通スタイルは定数にまとめ、後から絵柄を一括で変えられるようにしておく。
- 集約スクリプトの基準実装は `why-programmers-think-textbook-ja/scripts/figures/build_all.py`。新しい本の図版もこれに倣い、共通のパレット定数（`FIGURE_GUIDE.md` の固定パレット）とヘルパー関数（`head` / `rect` / `txt` / `ln` / `circ` / `poly` / `save`）を使い回す。座標や色を各図に直書きしない。
- 静的な SVG を手で持っている本を改修するときは、まず全図をこの集約スクリプトに移し、`figures/` の手書き SVG は再生成物（`assets/`）に置き換える。1 コマンドで全図を再生成できる状態を必ず残す。

## 非推奨

- `final-final-v2.png` のような曖昧な名前
- 本文と対応しない図番号
- 出典不明の画像
- 差し替え前の不要ファイルを残し続けること
